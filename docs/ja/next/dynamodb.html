DynamoDBを使用する場合

Global Setup/Deardown と Async Test Environment API を使用して、Jest は DynamoDB とスムーズに連携できます。

jest-dynamodb プリセットを使用する

Jest DynamoDB は、DynamoDB を使用してテストを実行するために必要なすべての設定を提供します。

  1. First, install @shelf/jest-dynamodb

    yarn add @shelf/jest-dynamodb --dev

  1. 次のように Jest の設定にプリセットを指定します。
{
  "preset": "@shelf/jest-dynamodb"
}
  1. jest-dynamodb-config.js を作成し、DynamoDB テーブルを定義します。

テーブルAPIの作成 を参照。

module.exports = {
  tables: [
    {
      TableName: `files`,
      KeySchema: [{AttributeName: 'id', KeyType: 'HASH'}],
      AttributeDefinitions: [{AttributeName: 'id', AttributeType: 'S'}],
      ProvisionedThroughput: {ReadCapacityUnits: 1, WriteCapacityUnits: 1},
    },
    // etc
  ],
};
  1. DynamoDBクライアントを設定します。
const {DocumentClient} = require('aws-sdk/clients/dynamodb');

const isTest = process.env.JEST_WORKER_ID;
const config = {
  convertEmptyValues: true,
  ...(isTest && {
    endpoint: 'localhost:8000',
    sslEnabled: false,
    region: 'local-env',
  }),
};

const ddb = new DocumentClient(config);
  1. テストを記述します。
it('should insert item into table', async () => {
  await ddb
    .put({TableName: 'files', Item: {id: '1', hello: 'world'}})
    .promise();

  const {Item} = await ddb.get({TableName: 'files', Key: {id: '1'}}).promise();

  expect(Item).toEqual({
    id: '1',
    hello: 'world',
  });
});

There's no need to load any dependencies.

詳細は ドキュメント を参照してください。
